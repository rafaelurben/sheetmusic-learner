# music timing

The metronome logic is run entirely on the frontend and is based on a start timestamp.

## metronome logic

`frontend/src/service/metronomeUtils.ts` contains utilities to calculate the timing of the metronome ticks based on
the sections of the piece containing the tempo (BPM), the time signature and the amount of bars using that config.

`frontend/src/service/metronomeService.ts` uses the "playlist" generated by the utilities to schedule the metronome
ticks using the Web Audio API. The start timestamp is used as a reference point to calculate the exact timing of
each tick. This allows precise playback.

## Decision: bpm

The bpm is defined as the number of beats per minute. I made the **decision** to use the time signature denominator as
the beat unit. For example, in a 4/4 time signature, the beat unit is a quarter note, so the bpm would indicate how
many quarter notes are played per minute. In a 6/8 time signature, the beat unit is an eighth note, so the bpm would
indicate how many eighth notes are played per minute.

## Decision: synchronization

Because browser timers are not perfectly accurate for security reasons (fingerprinting), `Date.now()` is not precise.
That means simultaneous playback on multiple clients will not be perfectly in sync, but it will be close enough for the
purpose of this project. Implementing a more complex synchronization mechanism would add too much complexity for this
project.
